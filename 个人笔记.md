
### 1. MyBatisPlus 使用的基本流程 (How)
要在项目中使用 MyBatisPlus, 需要完成以下五个步骤:
* **引入依赖**: 引入 MyBatisPlus 的 Spring Boot Starter 依赖.
* **配置扫描**: 在启动类上添加 `@MapperScan` 注解, 告诉框架 Mapper 接口的位置.
* **创建 Mapper**: 编写数据访问接口, 继承 `BaseMapper<T>` (其中 `T` 是实体类).
* **配置实体**: 编写实体类, 并通过注解 (如 `@TableName`) 声明它与数据库表的对应关系.
* **参数微调**: 在 `application.yml` 中配置可选参数 (如扫描包路径, 主键策略等).

### 2. MyBatisPlus 的核心原理 (Why)
解释了 MP 如何通过**反射 (Reflection)**免去开发者编写 SQL 的过程:
* **什么是反射**: Java 的一种机制, 像“X 光机”一样, 允许程序在**运行状态 (Runtime)**下动态获取任意一个类的结构 (类名, 属性, 注解等).
* **工作三步走**:
   1. **扫描**: 启动时找到所有实体类的字节码文件.
   2. **反射**: 动态读取类名 (映射为表名) 和属性名 (映射为列名).
   3. **映射**: 在内存中建立 Java 对象与数据库表的映射关系, 从而在执行增删改查时, 自动拼接并执行 SQL 语句.

### 3. 条件构造器 (Wrapper)

条件构造器主要用于构建复杂的 SQL 条件 (WHERE 部分) 和特殊的更新字段 (SET 部分). 

#### 核心分类
* **`QueryWrapper` 和 `LambdaQueryWrapper`**: 通常用来构建 `select`, `delete`, `update` 的 `where` 条件部分. 
* **`UpdateWrapper` 和 `LambdaUpdateWrapper`**: 通常只有在 `set` 语句比较特殊 (例如更新字段需要基于原本字段值进行运算) 时才使用. 
* **Lambda 表达式版本 (推荐)**: 尽量使用 `LambdaQueryWrapper` 和 `LambdaUpdateWrapper`, 避免在代码中硬编码字段名. 

#### 常用条件方法
* `eq`: 等于 (`=`)
* `ne`: 不等于 (`<>`)
* `gt` / `ge`: 大于 / 大于等于 (`>` / `>=`)
* `lt` / `le`: 小于 / 小于等于 (`<` / `<=`)
* `like` / `likeRight`: 模糊查询 / 右模糊查询
* `between`: 介于两者之间 (`BETWEEN a AND b`)
* `isNull` / `isNotNull`: 字段为空 / 不为空
* `in`: 在集合中 (`IN (a, b, c)`)

### 4. 自定义 SQL (Custom SQL)

当面对多表关联查询 (例如 `JOIN`) 或非常复杂的 SQL 时, 纯用 `Wrapper` 会很不方便 (因为单表 `Wrapper` 默认不支持直接拼接 `JOIN` 语法). 

MyBatisPlus 提供了**自定义 SQL** 功能, 允许我们自己编写多表关联的部分, 而把 `WHERE` 条件部分的构建依然交给 `Wrapper` 处理. 

#### 使用步骤
1. **在 Mapper 接口中定义方法**:
   使用注解 (或 XML) 编写主要的 SQL 语句 (如 `JOIN` 部分), 并在结尾使用 `${ew.customSqlSegment}` 占位符代表整个 `WHERE` 子句. 
   ```java
   @Select("SELECT u.* FROM user u INNER JOIN address a ON u.id = a.user_id ${ew.customSqlSegment}")
   List<User> queryUsersByAddressAndIds(@Param(Constants.WRAPPER) Wrapper<User> wrapper);
   ```
   *注意*: 参数必须使用 `@Param(Constants.WRAPPER)` 注解修饰, 其值即为 `"ew"`. 

2. **在测试类或 Service 中调用**:
   在使用多表查询时, `QueryWrapper` 中指定的字段名需要带上表别名 (例如 `u.id` 或 `a.city`), 因此一般不用 `LambdaQueryWrapper` 而采用普通的 `QueryWrapper`. 
   ```java
   @Test
   void testCustomSql() {
       List<Long> ids = List.of(1L, 2L, 4L);
       QueryWrapper<User> wrapper = new QueryWrapper<User>()
               .in("u.id", ids)
               .eq("a.city", "Beijing");
               
       List<User> users = userMapper.queryUsersByAddressAndIds(wrapper);
       users.forEach(System.out::println);
   }
   ```
